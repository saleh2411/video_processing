Chapter 10

Video Matting

Green-screen, but without a green screen. The pipeline is trimap (definitely-fg / definitely-bg / unsure) → opacity map (a real α value for each unsure pixel) → matting (composite onto a new background without colour bleed).

10   Video Matting

Original text (PDF page 67)

The purpose of video matting is taking the foreground of an image and insert it to a new background. The entire framework of video matting compromises:

Input: A 3-channel image ($I : \Omega \to R^3$ or denoted as $c: \Omega \to R^3$) and scribbles of the foreground ($\Omega_F \in \Omega$) and the background ($\Omega_F \in \Omega$). We can even add more information to the input (such as optical flow for better segmentation), so the input image does not necessarily belong to $R^3$.

Trimap: This is essentially a map of only 3 values (hence the name): 1 - we confidently know the pixel is foreground. 0 - we confidently know the pixel is background. Undecided value $\phi$ - we are not sure if the pixel is foreground or background.

Opacity map: It will fill the undecided region with values which are between 0 and 1.

Matting: This process takes as input an opacity map and a new background, and performs the final matting between the foreground of the original image and the new background. In regions where the opacity map is 0, the new background is inserted, while in regions where the opacity map is 1, the original foreground is preserved.

The question remains on what to insert to pixels where the opacity map was brtween 0 and 1. One possible way is to implement $\alpha I + (1 - \alpha)B$, but we won't do that. The reason is because in all those semi-transparent areas we will have a mix between the colors of the foreground and the background, which is not desired. For example, if we had a light hair on a red background, the color of the hair will be mixed with red.

Plain-English explanation

Matting is "extract the person, put them on a new background." Easy parts: clearly-inside-person and clearly-not-person pixels. Hard part: the boundary — wisps of hair, fuzzy edges — where a pixel is a mix of foreground and background colour.

Three stages:

  1. Trimap — every pixel labelled 1 / 0 / unsure (φ).
  2. Opacity map — each unsure pixel gets a real $\alpha \in [0,1]$ ("how foreground is it?").
  3. Matting — composite onto the new background without colour bleed from the old one.

Why "trimap" instead of just "mask"

A binary mask (foreground/background) is what we eventually want, but at fuzzy edges no binary answer is honest. The trimap admits "I don't know for these pixels — refine me" instead of guessing. Stage 2 then spends compute only where it matters (the narrow uncertain band), not the whole image.

The hair-on-red-background nightmare

A single pixel of light hair on a red shirt is, say, 60% hair + 40% red shirt. Its observed colour is a mix — pinkish-white. If you simply paste it on a new green background, the pinkish-white pixel doesn't match either pure hair OR pure green — it looks wrong. The third stage (matting) recovers the PURE hair colour and re-mixes properly.

Where the scribbles are critical

The whole algorithm is bootstrapped from the user's scribbles. Sparse scribbles in well-chosen locations do the trick: at least one scribble per distinct colour region of foreground/background. A scribble for a person's blonde hair and another for brown skin = good; one stroke straddling both = bad.

10.1 Trimap Estimation

Original text (PDF pages 68–70)

If we had a light foreground and dark background, the most simple approach would be creating a histogram, dividing the values to bins and finding a threshold for which bins belong to the background. However, we are working with at least 3-channel images, so making for example 10 bins in each dimention, we will have in total 1000 bins which is a lot.

We will, therefore, work with what is called Kernel Density Estimation (KDE). This is a method to estimate a distribution from a finite sample set. What we have is the color value $c \in R^3$ for every pixel, and we want to find the color distribution of foreground and background. The scribles themselves indicate we have conditional probability: we have a color given the scribble is foreground or background. More formally: $\forall x \in \Omega_F : c(x) \in f_{C|F}$ and $\forall x \in \Omega_B : c(x) \in f_{C|B}$.

We will start with an example in 1-D. Usually, if we want to create a histogram, we would divide the axis to bins and count how many pixels we had at every bin. Instead, we would draw around each pixel a "bell".

We estimate:

$$ f(c|F) \propto \sum_{x \in \Omega_F} K(c, c(x)) $$

The same will be done with the background. This is sometimes called Parzen window. Usually the function K is: $K(c, c') \propto e^{-\frac{\|c - c'\|_2^2}{2\sigma^2}}$. An ideal histogram would look like: [two non-overlapping bumps]

However, in most cases, we will have a very large overlap. The complexity would be for calculating $f(c|F)$ is $O(|\Omega| \cdot |\Omega_F|)$ since K gets two parameters c and c'. This is too large, and as a result we will do something else. We will assume for now we have calculated them, and later introduce a practical approach for them.

We will develop the equations:

$$ P(F|c) = \frac{f(c|F) P(F)}{f(c)} = \frac{f(c|F) P(F)}{f(c|F) P(F) + f(c|B) P(B)} $$
$$ P(B|c) = \frac{f(c|B) P(B)}{f(c)} = \frac{f(c|B) P(B)}{f(c|F) P(F) + f(c|B) P(B)} $$

The main parameters unkown to us here are P(F) and P(B). In worst case, we can assume uniform distribution: $P(F) = P(B) = \frac{1}{2}$, $\implies P(F|c) = \frac{f(c|F)}{f(c|F) + f(c|B)}$.

Now we will do distance maps similar to video colorization. We had a path $p \in \Omega$ but now:

$$ L(p) = \int_p |\langle W(p(t)), \dot{p}(t) \rangle| dl, \qquad W(x) = \nabla_x p(F|c(x)) $$

We have taken this function because it will be consistently larger in the foreground and smaller in the background. Next:

$$ x, y \in \Omega: d_L(x, y) = \min_{p \in [x, y]} L(p) $$

Where we go over all the points p from x to y and find the minimum. Next:

$$ D_F(x) = d_L(x, \Omega_F) = \min_{y \in \Omega_F} d_L(x, y), \qquad D_B(x) = d_L(x, \Omega_B) = \min_{y \in \Omega_B} d_L(x, y) $$

This will eventually create two regions: the first region would be for the foreground and the second would be for the background.

Within each region, pixels will have similar distance values, whereas crossing between regions leads to a sharp change in distance. Consequently, in each distance map, one region will exhibit relatively small values while the other will have larger values. By comparing the two distance maps, we can determine whether a given pixel belongs to the foreground or the background:

$$ V_F = (x \in \Omega: D_F(x) < D_B(x)), \quad V_B = (x \in \Omega: D_F(x) > D_B(x)) $$

And we can also define the boundary: $\Delta = (x \in \Omega: D_F(x) = D_B(x))$.

Usually, this set will be empty because the probabilty for the distance to be equal is very low. As a result, the boundary can be found using interpolation. Now, we will create a narrow band around the boundary:

$$ B_p(\Delta) = \bigcup_{x \in \Delta} B_\rho(x) $$

Where $B_\rho(x)$ is a ball around x with radius $\rho$. We can also write this as:

$$ x \in B_\rho(\Delta) \iff \exists y \in \Delta : \|x - y\| <= \rho $$

Finally, the Trimap will be:

$$ \forall x \in V_F \setminus B_\rho(\Delta): \alpha_T(x) = 1 $$
$$ \forall x \in V_B \setminus B_\rho(\Delta): \alpha_T(x) = 0 $$
$$ \forall x \in B_\rho(\Delta): \alpha_T(x) = \phi $$
Plain-English explanation

Step 1 — KDE. 3D histograms would have 1000 bins (mostly empty). Instead, place a soft Gaussian "bell" around each scribble pixel. Sum the bells for foreground → $f(c|F)$. Same for background.

Step 2 — Bayes. $P(F|c) = f(c|F) / (f(c|F)+f(c|B))$ (uniform prior).

Step 3 — Geodesic distance using $P(F|c)$ as the underlying field. Compute distance to FG scribbles ($D_F$) and to BG scribbles ($D_B$).

Step 4 — Classify. $D_F < D_B$ → foreground; otherwise background. The set where they're equal = the boundary $\Delta$.

Step 5 — Trimap. Far from boundary → confident 0/1. Within radius ρ of boundary → "unsure" (φ). The narrow uncertain band is what the opacity map handles next.

KDE in plain words

Think of each scribble pixel as putting a little colored bell-curve "puff" on a colour map. Sum all the puffs from foreground scribbles → you have a smooth surface that's high in colour regions the foreground occupies, low elsewhere. Same for background. To classify a new pixel, look up its colour on both surfaces and pick the one that's higher.

Why we don't just use colour alone

Sometimes foreground and background have OVERLAPPING colours (think a man in a brown jacket against a brown sofa). The colour distributions overlap heavily, so just $P(F|c)$ would say "I dunno, could be either." Geodesic distance breaks the tie using spatial proximity through edges: even if colour is ambiguous, the pixel is closer (via edges) to one scribble than the other.

Why ρ matters

ρ defines the width of the "unsure" band. Too small → boundary region is one pixel wide, opacity map can't capture soft hair. Too big → most of the image is "unsure" and the opacity map has too much work to do. Typical ρ ≈ 5–15 pixels.

10.2 Trimap Refinement — Opacity map

Original text (PDF pages 71–72)

Let's draw only the area that is interesting for us:

At this stage, we now have a much richer set of data. Previously, we only had a few scribbles indicating foreground and background, which were the sole sources of information for the entire image, despite its large color variability. Now, our focus is on a specific region with much more limited color variation. We have a lot more samples of the foreground and background on the boundary.

Thus, we will estimate again the distribution functions for refinement. And since we have a curve, we can put a coordinate system and measure the length of the curve. We will denote that as $\tau$. Every point we can denote as $(\tau, c(x))$. Since the boundary in very narrow, this problem is nearly 1-D. Now we can develop the equations:

$$ \forall x \in B_\rho(\Delta): \quad \alpha() = \frac{w_F(x)}{x_F(x) + w_B(x)} $$
$$ w_F(x) = \tilde{D}_F(x)^{-r} \tilde{P}_F(x), \qquad r \in (0, 2] $$

Here $\tilde{P}_F(x)$ is computed based on previous step. The equation of $w_F(x)$ was chosen intuitively in the paper.

Plain-English explanation — opacity map

The first KDE used scribbles all over the image to model FG/BG colour distributions. But colour statistics near the boundary can differ from the bulk. So re-estimate distributions using just pixels right next to the boundary — much more reliable locally. Then combine local probability $\tilde P_F$ with geodesic distances to assign each unsure pixel a continuous $\alpha \in [0,1]$.

10.3 Matting

Original text (PDF pages 72–73)

Now we are given:
B - new background
$\alpha$ - opacity map
c - original image.
As said earlier, we could do: $J(x) = \alpha c(x) + (1 - \alpha(x)) B(x)$.

This will be done on the foreground and background, but not on the boundary. That is because the colors will mix and we even may get a halo around the foreground. We will do something else then. Suppose we have the point x:

We can write down the value of the pixel as:

$$ c(x) = \alpha f(x) + (1 - \alpha(x)) b(x) $$

Where in this case f(x) and b(x) are some unkown foreground and background values respectively. Our goal would be to estimate them. This is an ill-problem (one equation with 2 unknowns), but we have a lot of colors around the pixel that we know they are foreground and background. We will open a window around the pixel, and claim that there are two pixels in the window sutisfying the equation above.

On a very small area this assumption most of the time works. What we will do is each time take two pixels in the window, compute the value according to the equasion earlier, substract c(x), calculate the norm and find those which minimize the result:

$$ (x_F^*, x_B^*) = \arg\min_{\substack{x_F \in N(x) \cap \{x:\alpha(x)=1\} \\ x_B \in N(x) \cap \{x:\alpha(x)=0\}}} \|\alpha(x) c(x_F) + (1 - \alpha(x)) c(x_B) - c(x)\|^2 $$

With that we can take the color in $x_F^*$ as the value of the foreground object than c(x):

$$ J(x) = \alpha c(x_F^*) + (1 - \alpha(x)) B(x) $$
Plain-English explanation — matting

Naive blend $J = \alpha c + (1-\alpha) B$ fails: at a fuzzy hair pixel, $c$ already mixes hair + the OLD red background. So the result drags the old red onto the NEW background.

Fix: recover the PURE foreground colour $f(x)$. We have one equation with two unknowns ($f$, $b$). Assumption: in a small window, $f$ matches some confident-FG pixel $x_F^*$ and $b$ matches some confident-BG pixel $x_B^*$. Search the window for the pair that best explains the mixed pixel. Composite using the pure $c(x_F^*)$ instead of $c(x)$.

10.4 KDE — Kernel Density Estimation

Original text (PDF pages 73–74)

Now we are only left to calculate KDE efficiently. We want to calculate $\forall x \in \Omega$:

$$ f(x) = \sum_{y \in \Omega'} k(x, y) w(y) $$

The computation efficiency here is $O(|\Omega| \cdot |\Omega'|)$. We assume the kernel is low-rank. To understand this, lets assume we have:

$$ f_{n \times 1} = K_{n \times m} w_{m \times 1} $$

Low rank means: $\text{rank}(K) = p \ll m, n$.

The computation efficiency here is O(mn). We can write this as a sum of matrices with rank 1:

$$ K = \sum_{i=1}^p \phi_i \psi_i^T = \begin{pmatrix} | & \cdots & | \\ \phi_1 & \cdots & \phi_p \\ | & \cdots & | \end{pmatrix} \begin{pmatrix} -~\psi_1^T~- \\ \vdots \\ -~\psi_p^T~- \end{pmatrix} = \Phi_{n \times p} \Psi_{m \times p}^T $$

The rank of this matrix is maximum p. Now we will insert this to previous equation: $f = \Phi \Psi^T w$.

The computation time of $\Psi^T w$ is $O(pm)$, where p is very small. If we denote $a = \Psi^T w$, then $f = \Phi a$ will take $O(pn)$. In total we will have $O(p(m+n))$. Insead of quadratic complexity, we have linear complexity. If we say that our kernel $K(x,y)$ is low rank, then: $p \ll |\Omega|, |\Omega'|$.

we can write it as:

$$ K(x, y) = \sum_{i=1}^p \phi_i(x) \psi_i^T(y), \qquad a_i = \sum_{y \in \Omega'} \Psi_i(y) w(y), \qquad f(x) = \sum_{i=1}^p a_i \phi_i(x) $$

The calculation of $a_i$ takes $O(|\Omega'|p)$, the calculation of the last equation also takes $O(|\Omega|p)$ - in total $O((|\Omega| + |\Omega'|)p)$. Thus there is a lot of consideration when choosing the kernel and to pick one which is low rank. Gaussian kernel is for example infinite-rank, but if we take its singular values we will see they decay to 0. We will not discuss this in the course, but the methods for factorizing kernels is called Fast multiple method.

Plain-English explanation — making KDE fast

Problem: KDE at every pixel = sum over every scribble pixel = $O(NM)$. Too slow for 1MP images.

Trick — low-rank kernel. If $K(x,y) = \sum_{i=1}^p \phi_i(x) \psi_i(y)$ with small $p$, the sum factors:

  • $a_i = \sum_y \psi_i(y) w(y)$ — depends only on y. Compute ONCE in $O(Mp)$.
  • $f(x) = \sum_i a_i \phi_i(x)$ — for each pixel, just p multiplications.

Total: $O((N+M)p)$ — linear, not quadratic. Gaussians are infinite-rank but their singular values decay, so a small effective rank works. Fast Multipole Method exploits this.

A general lesson — low-rank everywhere

The same "low-rank trick" appeared in Chapter 5 (texture as low-rank state-space). It's one of the most powerful ideas in numerical computing: if a matrix has effective rank $p \ll \min(m, n)$, you can usually replace an O(mn) operation with O((m+n)p). Look for opportunities everywhere.

Rule of Thumb
  • Scribble inside each colour region of FG and BG. One per region is the minimum; more is safer.
  • Boundary band radius ρ ≈ 5–15 pixels. Smaller = sharper but misses hair; larger = smoother but expensive.
  • Use BOTH local KDE refinement and geodesic distance for the opacity map. Either alone is fragile; combined they're robust.
  • Don't blend the original pixel directly at the boundary — recover pure foreground (using neighbour windows) and composite that on the new background, to avoid colour bleed.
  • Add optical flow as an extra channel in video. It separates moving objects from static background better than colour alone.
  • For real-time use, swap the iterative geodesic computation for fast sweeping (Ch.9).
