Take a shaky video of a person walking and output a stabilized clip of them — re-composited onto a brand-new background, with a tracking box following them through every frame. All with classical computer vision. No deep learning allowed.

The pipeline

Four blocks, in order

Each block's output feeds the next. Files are named <stage>_ID1_ID2.avi. Open workflow.html for the interactive data-flow diagram.

1

Stabilization

Estimate how the camera moves between consecutive frames, smooth that trajectory, then warp each frame to cancel the shake. The walking subject still moves — only the camera jitter is removed.

Feature detect + matchOptical flowAffine / homographyTrajectory smoothing
in: INPUT.avi  →  out: stabilize_ID1_ID2.avi
2

Background Subtraction

Model the static scene and pull out the moving person. Produces two things: the person cut out on a black background, and a binary mask. Clean noisy edges with morphology (open/close) and pick the largest blob.

KNN / MOG2Color-space statsThresholdMorphologyConnected components
in: stabilize.avi  →  out: extracted_ID1_ID2.avi + binary_ID1_ID2.avi
3

Matting

Build a soft alpha matte (0..1) so the person's edges blend naturally — no harsh cut-out look. Composite the person over the new background image: out = α·foreground + (1−α)·newBg. Inputs are the background image, the stabilized video, and the binary mask.

Trimap from maskAlpha estimationForeground / background colorsAlpha compositing
in: background.jpg + stabilize.avi + binary.avi  →  out: matted_ID1_ID2.avi + alpha_ID1_ID2.avi
4

Tracking

Automatically lock onto the single walking person and draw a rectangle around them in every frame. Record each box as [ROW, COL, HEIGHT, WIDTH]. Graded against secret ground-truth boxes, so accuracy of the box matters.

Particle filterMean-shift / CAMShiftTemplate matchMask bounding-box
in: matted.avi  →  out: OUTPUT_ID1_ID2.avi + tracking.json

Tips that move the grade

What actually matters

🎯

Don't overfit the secret video

You may study color spaces offline and hard-code constants, and use first-frame scribbles. But the second test video has the same person in different clothes on a maybe-different background. Tune for robustness, not just INPUT.avi.

🔪

Keep stabilization sharp

Interpolation blur quietly wrecks the matting stage — fuzzy edges = bad alpha. Black border regions from warping are explicitly fine.

⏱️

Respect the 20-minute wall

Runs are aborted at 20 min on the lab machines and runtime is part of the grade. Avoid per-pixel Python loops — vectorize with NumPy, downscale where you can.

📁

Relative paths only

Use os.path to build paths. No C:\... absolutes — the grader runs on Ubuntu 22.04 from the repo root.

📝

The report is graded

Explain each stage in the PDF. Any function "too specific" (one that does a whole stage for you) or any non-class algorithm must be explained, with source — or you lose points.

🧹

Clean morphology pays off

Most background-subtraction grade comes from a clean mask: open to kill speckle, close to fill holes, keep the largest connected component as the person.

🚫

Hard rules: No deep learning / deep networks. No single built-in library function that solves an entire main block (stabilize, track, background-sub, or matting) end-to-end. opencv functions are allowed if you explain how they work in the report.

🪜

The "way out": a full solution built on opencv-contrib trackers/subtractors — roughly ±10 lines per stage. It caps your project grade at 70/100. Use it only as a safety net.

Don't start from scratch

Reuse code from ex1 / ex2 / ex3

Two stages are nearly drop-in from your own course exercises (your implementations — not banned built-ins). Two stages have no prior code and must be built fresh.

1

Stabilization  →  ex2 lucas_kanade.py  near drop-in

The whole file. lucas_kanade_faster_video_stabilization_fix_effects() is a complete stabilizer with video I/O, backed by build_pyramid, lucas_kanade_step, warp_image / warp_image_remap, lucas_kanade_optical_flow and the faster_* variants. Its inner crop_resize trims warp borders — that directly answers project FAQ #1.

2

Background Subtraction  →  no reusable code

Nothing in ex1–3 covers this. Build fresh: background model (KNN / MOG2 or color-space stats) → threshold → morphology → largest connected component. Borrow only the gradient helpers below for edge cues.

3

Matting  →  no reusable code

Also fresh. Trimap from the binary mask → alpha estimation → composite. Reuse create_grad_x_and_grad_y (ex1) for boundary detection in the trimap band.

4

Tracking  →  ex3 particle_filter.py  near drop-in

The whole file: predict_particles, compute_normalized_histogram, sample_particles, bhattacharyya_distance, compute_weights_and_cdf, show_particles. Already dumps per-frame state to JSON — adapt that into tracking.json.

Starting point

main.py skeleton

A timing-aware structure that matches the required outputs and JSON keys.

import os, json, time
import cv2, numpy as np

ID1, ID2 = "213851579", "213310485"
ROOT = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
IN  = os.path.join(ROOT, "Input")
OUT = os.path.join(ROOT, "Outputs"); os.makedirs(OUT, exist_ok=True)
def p(name): return os.path.join(OUT, f"{name}_{ID1}_{ID2}.avi")

t0 = time.time()
timing = {}

# 1. stabilize INPUT.avi  -> stabilize_*.avi
stabilize(os.path.join(IN,"INPUT.avi"), p("stabilize"))
timing["time_to_stabilize"] = time.time()-t0

# 2. background subtraction -> extracted_*.avi + binary_*.avi
background_subtract(p("stabilize"), p("extracted"), p("binary"))
timing["time_to_binary"] = time.time()-t0

# 3. matting -> matted_*.avi + alpha_*.avi
matting(p("stabilize"), p("binary"), os.path.join(IN,"background.jpg"),
        p("matted"), p("alpha"))
timing["time_to_alpha"]  = time.time()-t0
timing["time_to_matted"] = time.time()-t0

# 4. tracking -> OUTPUT_*.avi + tracking.json
boxes = track(p("matted"), p("OUTPUT"))
timing["time_to_output"] = time.time()-t0

json.dump(timing,   open(os.path.join(OUT,"timing.json"),   "w"))
json.dump(boxes,    open(os.path.join(OUT,"tracking.json"), "w"))
