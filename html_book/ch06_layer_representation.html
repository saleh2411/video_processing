Chapter 6

Layer Image Representation

An animation is a sandwich of layers: background, characters, effects. Each layer has its own motion. Given the final composed video, can we recover the original layers? Yes — by alternating Lucas–Kanade motion fits with K-means clustering in the 6-D space of affine motions.

Original text (PDF page 45)

In this lecture we will go over two topics: Layered image representation and background subtraction. What is common to both of them is that we are given a signal (a video or a signal as a function of time), and we want to divide it into basic components.

Layer image representation - introduction

Usually in animated videos, each frame is divided into layers: the first layer is the background, the second contains the characters, etc. Given these layers, we can generate the final video. However, we are interested in the reverse process—given the video, we want to decompose it into different layers.

By using a very basic transformation (for example, Affine), we will see that it is quite easy to model each layer in the video. Most of the time, we can estimate the behavior of very complex scenes using a small number of coherent motions. Usually, a small neighborhood of pixels moves in the same direction rather than each pixel moving independently.

background substruction - introduction

Given a video, we want to determine for every pixel whether it belongs to a static object or a dynamic object. One simple approach is to observe the intensity of the pixel as a function of time. If it does not change much, we can assume it belongs to a static object.

Plain-English explanation

Movies are built like sandwiches: background layer, then characters on top, then effects. The compositor stacks them, you see the final frame. The reverse problem — given the final video, recover the original layers — is hugely useful for editing, object removal, background swap.

Key assumption: each layer moves coherently — every pixel in a layer follows the same simple transformation (typically affine, just 6 numbers). So instead of millions of unknowns, you have just a few "motion groups."

6.1 Layer image representation

Original text (PDF pages 45–46)

We assume:

$$ I_0(x, y) = E_0(x, y) $$
$$ I_1(x, y) = I_0(x, y)(1 - \alpha_1(x, y)) + E_1(x, y) \alpha_1(x, y) $$
$$ I_k = I_{k-1}(1 - \alpha_k) + E_k \alpha_k $$

$E_k$ - are the layers (where $I_0$ is the background layer).
$I_k$ - are the images.
$\alpha_k$ - are the masks.

The second equation tells us that the image $I_1$ is a combination of the previous image and the current layer. This is done using the mask $\alpha_1$ (for example, a binary mask), which determines whether we take the pixel from the current layer or from the previous image. The order of the images determines which image overlaps the other.

The full synthesis process they came up with is the following:

This diagram illustrates the equations described earlier, where the dashed block is duplicated according to the number of layers. The assumption is that when we observe $I_1$, it was generated by the two layers $E_0$ and $E_1$.

To generate $I_0$, we start with a background layer $E_0$, warp that layer using a vector field $V_0$ that describes the motion, optionally add a differential image $D_0$ if needed, and obtain $I_0$. The same process is applied to the next layer $E_1$; however, in this case, we also warp with respect to its mask $\alpha_1$ and then combine the previous image with the current layer using the mask to obtain $I_1$.

We have two consecutive frames - blue and green. $V_0$ and $V_1$ are the motions and change for every timestep. The layers $E_0$ and $E_1$, however, remain constant on every timestep.

What we have shown so far is the synthesis process. What we want, however, is the analysis process: given $I_k$ and the number of layers, we want to determine, for each pixel, which layer it belongs to. In other words, we aim to reverse the process to recover the layers $E_0$ and $E_1$, while simultaneously estimating the motion fields $V_k$ and the masks $\alpha_k$.

Defining the problem: Given a pair of frames $I_1$ and $I_2$, and the number of layers k, find the segmentation of the motion between the pair of images to k Affine motions.

Plain-English explanation

Alpha compositing. $\alpha$ is the mask, a number between 0 and 1 per pixel:

  • $\alpha = 0$: fully transparent — keep what's underneath.
  • $\alpha = 1$: fully opaque — show this layer.
  • In between: blend.

The recursive formula $I_k = I_{k-1}(1-\alpha_k) + E_k\alpha_k$ is the workhorse of every compositor on Earth — from Photoshop to After Effects to film VFX.

Forward problem (synthesis) is easy: layers + masks + motions → final video.

Inverse problem (analysis) is hard: final video → layers + masks + motions. Make it tractable by assuming each layer's motion is just one affine transform (6 numbers).

Original text — Algorithm (PDF page 47)

Algorithm:

1. L-K on the regions.
2. K-means.
3. Go to 1 until convergence.

We have no a-priori knowledge of which regions move in which direction, so we first run LK on different regions of the image using observation windows. We can use any motion transformation, but for now, let's assume an affine transform. For each window, we obtain a vector of parameters $V \in R^6$, which maps and transforms the window in image $I_1$ to its corresponding position in image $I_2$.

Next, we run K-means clustering in the space $R^6$ to identify 2-D affine motions that are consistent across the image plane. By clustering windows with similar motion, we can recover the original layers. This process is repeated over several iterations until the algorithm converges to a stable solution. At each iteration, we select random windows to perform the pixel-level classification. The algorithm will naturally cluster the most dominant motions in the image.

K-means

Given a set of points $x \in R_d$, separate them into groups of k. We will define:

$$ \text{Encode}: E^d \to [1, ..., k] $$
$$ \text{Decode}: [1, ..., k] \to E^d $$
$$ \text{Distortion} = \sum_{i=1}^n \|x_i - \text{Decode}(\text{Encode}(x_i))\|^2 $$

Our goal is to minimize the distortion. For example, if we have the following set of points, we want to find their centers:

We will denote those centers (sometimes called prototypes) as: $C_j = \text{Decode}[j]$ and we got:

$$ \text{Distortion} = \sum_{i=1}^n \|x_i - C_{\text{Encode}(x_i)}\|^2 $$

Our goal is given dataset x and the number of clusters k, we want to find the prototypes that minimize the distortion.

Algorithm K-Means:

1. Initialize $c_1, ..., c_k$
2. Assign $x_i$ to the closest $c_j$
3. $c_j = \frac{1}{|s_j|} \sum_{i \in s_j} x_i$
4. Return to section 2.

This is an iterative algorithm. We first initialize the prototypes randomly. Then, for each data point, we assign the closest prototype. Next, we update the prototypes as the average of all data points assigned to the same prototype. This process is repeated until convergence.

We assigned each datapoint $x_i$ to it's closest prototype $c_j$ because it will have the smallest euclidean distance: $\|x_i - c_l\|^2 > \|x_i - c_j\|^2$. This assignment will minimize the distortion. After that, we take the average and this is because if we differentiate the distortion and compare to 0:

$$ \frac{\partial \text{Distortion}}{\partial c_j} = \frac{\partial}{\partial c_j} \sum_{i=1}^n \|x_i - C_j\|^2 = -2 \sum_{i \in s_j} \|x_i - c_j\| = 0 \to c_j = \frac{1}{|s_j|} \sum_{i \in s_j} x_i $$

Thus, $c_j$ minimize distortion in square error model. Notice after step 2 we right away go to step 3 and won't apply step 2 a second or a third time. That is because after we do the second step we are already optimal so there is no need to do it twice.

Notice that the larger the value of k, the smaller the distortion will be. Ideally, if k equals the number of pixels, the distortion would be zero. There are algorithms based on the Minimum Description Length (MDL) principle that aim to find the optimal value of k for clustering. However, this is out of the scope for this course so we won't dive deeper into it.

Plain-English explanation — LK + K-means loop

Step 1. Run LK on many random windows. Each window gives a 6-number affine motion. So each window becomes a point in $\mathbb{R}^6$.

Step 2. Cluster in motion space. Windows whose motions are similar end up in the same cluster — each cluster represents one "layer" with one global motion.

Step 3. Repeat. Now you know which pixels belong to which layer, so re-fit LK per layer more accurately, re-cluster, etc.

K-means in 4 lines: drop k random centres → each point joins its nearest centre → each centre moves to its members' mean → repeat. Always reduces distortion, always converges (but depends on initialization).

Why this chicken-and-egg works

You need to know which pixels share a layer before you can fit a good per-layer motion. But you need a per-layer motion to know which pixels share a layer. The fix is "EM-style" iteration: assume one, solve the other, swap, repeat. Each iteration provably decreases the total distortion, so convergence is guaranteed (though it may land in a local minimum).

Picking k — the hardest part

k is the number of layers. Too small = motions get lumped together (background + slow car); too large = single objects get split. The PDF mentions MDL (Minimum Description Length) as the principled choice, but in practice people try k = 2, 3, 4, 5 and pick the result that "looks right." The MDL idea: balance the distortion (lower with bigger k) against a penalty for using more clusters.

Why "random windows" each iteration?

It's a stochastic shortcut. Running LK on every possible window of the image is expensive. Random sampling gives a noisy but unbiased estimate fast. Over many iterations, the noise averages out.

Where you've seen alpha compositing before

Every modern graphics pipeline. PNG transparency, Photoshop layers, CSS opacity, video editing tools — they're all the same formula $I = E \alpha + I_{prev}(1-\alpha)$. The "secret sauce" of this chapter is using that formula as both the synthesis model (compose frames from layers) and the analysis target (recover layers from frames).

Rule of Thumb
  • Start with k = 2 or 3. Most real videos have a background + 1-2 dominant moving things. Add layers only if you see clear motion you're missing.
  • Affine is usually enough for layer motion. Use homography only when a layer has clearly planar perspective (a building wall while panning).
  • Random sub-sampling of windows is good practice — way faster than dense LK and almost as accurate after a few iterations.
  • If K-means keeps returning empty clusters, your initialization is bad. Use k-means++ (seeds chosen far apart) instead of pure random.
  • Always run 2–3 random restarts of K-means and keep the lowest-distortion result.
  • Mask soft α matters at object edges. A hard binary mask gives jagged composites; a soft mask blends naturally (Ch.10 takes this much further).
