Chapter 5

Texture Synthesis

Generate endless frames of a flickering candle, swirling smoke, or rippling water that look like the source clip but never repeat. The trick: assume the texture is the visible output of a hidden low-dimensional state-space process. Recover that process with SVD, then drive it with random noise.

5   Texture synthesis

Original text (PDF page 41)

We will start by giving the bottom line of the algorithm and then develop the equations. In order to process a signal in time, we have learned we can use Kalman filter for that. However, Kalman won't be given to us and we will instead try to estimate it's parameters. We are given a series of frames:

Kalman assumes we have the measurements and the states. They assumed:

$$ x_{t+1} = A x_t + B v, \qquad y_t = C x_t + w $$

Where v is a random noise vector. A and $x_t$ are not given, and we aim to estimate them from the observations $y_t$. In some sense, texture can be viewed as a process in which we use past states to estimate the current state. To prevent the process from being purely deterministic, we introduce noise. This can be visualized as follows:

We define a state space in which the texture evolves over time along a trajectory. Each state along this trajectory is mapped to an image y. The mapping is defined by the second equation introduced earlier: we apply the matrix C and add noise w, which makes the generated image y appear more natural. Thus, the first equation represents the motion model in the state space, while the second equation describes the transformation from the state to the observed image, which is the final output we observe.

Plain-English explanation

Goal: after seeing a short clip of e.g. a flame, generate new frames that look like the same flame but aren't a literal copy.

Hidden-state insight: at any moment a "texture frame" is governed by a small hidden state $x_t$ (50 numbers, say). The state drifts over time with a simple linear rule + noise. Each state $x_t$ is rendered as an image $y_t$ via another linear map + noise. Learn $A$ and $C$ from a sample clip, then drive the state forward forever to generate novel texture.

"Dynamic texture" — what counts?

Anything that looks similar over time but is not literally repeating: ocean waves, flames, escalators, swaying grass, smoke, escalator handrails, falling water. The visual statistics are stationary even though individual frames differ. Anything with a recognisable object (a face walking by) is NOT a dynamic texture and this method won't work.

Why noise is essential

If the model were $x_{t+1} = Ax_t$ with no noise, after enough steps the state either dies (eigenvalues < 1), explodes (eigenvalues > 1), or cycles forever (eigenvalues on unit circle). Real textures don't do any of those — they wander. Noise $v(t)$ keeps the trajectory exploring while $A$ keeps it on the manifold of plausible textures.

The two noise sources, restated

  • $v(t)$ — drives the hidden state. This is what makes flames flicker, not repeat.
  • $w(t)$ — pixel noise on the final image. This is what hides the low-rank seams and gives a "real camera" feel.
Original text (PDF pages 42–44)

In order to solve the problem, we represent each frame y as a long vector. If there is a periodic phenomenon between the frames, then constructing a matrix from these frame vectors will result in a matrix whose rank is much smaller than the number of pixels. There are several solutions to this problem. For example, we could apply wavelet decomposition or DCT decomposition, but it has been found that SVD decomposition works best in practice. In other words, the solutions for $x_t$ correspond to the eigenvectors of the matrix described earlier.

We can estimate the process of the texture in time as an ARMA model:

Auto Regressive: $x_t = C + \sum_{i=1}^P p_i x_{t-1} + \epsilon_t$

Where C is constant, $p_i$ are the parameters and $\epsilon_t$ is the noise. This can also be written with vectors and a matrix:

$$ \begin{pmatrix} x_1 & \cdots & x_5 \\ x_2 & \cdots & x_6 \\ \vdots & \cdots & \vdots \end{pmatrix} \begin{pmatrix} p_1 \\ p_2 \\ \vdots \end{pmatrix} = \begin{pmatrix} x_6 \\ x_7 \\ \vdots \end{pmatrix}$$

Moving Average: $x_t = \mu + \sum_{i=1}^P \theta_i \epsilon_{t-1} + \epsilon_t$

Where $\mu$ is the mean value, $\theta_i$ are the parameters and $\epsilon_t$ are the noise.

Let's go back to our problem: we are given a series of images $(I(t))_{i=1}^T$ and we observe a series of images:

$$ y(t) = I(t) + w(t) $$

Let's assume we can model the texture in time with:

$$ x(t + 1) = A x(t) + B v(t), \qquad y(t) = C x(t) + w(t) $$

Where $x \in R^n (n \sim 50), y \in R^m (m \sim 200 \times 300)$. There are two noise sources: v(t) is for creating the texture (otherwise we will get a deterministic model which gives a constant behavior), and the second is the noise in the image itself to get a natural look.

To solve the problem, we need to define the error function we need to minimize. One way would be ML estimation: given y(1)...y(T), we would like to find:

$$ \hat{A}, \hat{B}, \hat{C}, \hat{q} = \arg\max_{A,B,C,q} \log P(y(1) \cdots y(T)) $$

s.t. the equations and $v(t) \sim q$ (q is a distribution). However, we will use Minimum Mean Error:

$$ \hat{A}, \hat{B}, \hat{C}, \hat{q} = \lim_{t \to \infty} E(\|y(t + 1) - \hat{C}\hat{x}(t + 1)|t\|) $$

However, this is also considered a hard problem to solve, so they came up with another approach (under some assumptions) to the problem:

$$ x(t + 1) = A x(t) + B v(t), \qquad y(t) = C x(t) + w(t) $$

However, $v(t) \sim N(0, Q)$ and $w(t) \sim N(0, R)$ and the initial condition of x is assumed to be $x_0$. There is not a single solution to this because if A, B and C are the solution, then given T an invertable matrix, we could replace $A = TAT^{-1}$, $Q = TQT^{-1}$, $C = CT^{-1}$ and $x_0 = T x_0$ and get an equivalent solution. In our solution we will choose one candidate from the set of solutions.

WE will use SVD decomposition to solve the problem. WE define:

$$ Y_1^\tau = [y(1) \cdots y(T)] \in R^{m \times \tau}, \tau > n $$
$$ X_1^\tau = [x(1) \cdots x(T)] \in R^{m \times \tau}, \tau > n $$
$$ W_1^\tau = [w(1) \cdots w(T)] \in R^{m \times \tau}, \tau > n $$

In matrix form: $Y_1^\tau = C X_1^\tau + W_1^\tau$, $C \in R^{m \times n}$.

We will demand $C^T C = I$, where I would be called the Identity Motion. This will strict the columns will be orthonormal and we chose this constraint to get a single solution from the set of all solutions. From SVD decomposition we will get:

$$ Y_1^\tau = U \Sigma V^T, \quad U \in R^{m \times n}, \Sigma \in R^{n \times n}, V \in R^{n \times \tau} $$

We assumed $w \sim N(0, R)$ so we search $\hat{C}, \hat{x}$ that minimize $\|w_1^\tau\|_F$. We will get:

$$ \hat{C}(\tau) = U, \qquad \hat{x}(\tau) = \Sigma V^T $$

To calculate $\hat{A}$ we need to solve:

$$ \hat{A}(\tau) = \arg\min_A \|x_1^\tau - A x_0^{\tau - 1}\| $$

Where: $X_0^{\tau-1} = [x(0) \cdots x(\tau - 1)]$, $X_1^\tau = [x(1) \cdots x(\tau)]$. The solution will be:

$$ \hat{A}(\tau) = \underbrace{\Sigma V^T D_1}_{X_1^\tau} V (V^T \underbrace{D_2 V)^{-1} \Sigma^{-1}}_{(X_0^{\tau-1})^{-1}} $$

Where:

$$ D_1 = \begin{pmatrix} 0 & 0 \\ I_{\tau-1} & 0 \end{pmatrix}, \qquad D_2 = \begin{pmatrix} I_{\tau-1} & 0 \\ 0 & 0 \end{pmatrix}, \qquad I_{\tau-1} \in (\tau-1) \times (\tau-1) $$
Plain-English explanation

Frames as vectors. Stack every frame's pixels into a single long column. Put all the columns side by side into a giant matrix $Y$. If the texture is low-dimensional, $Y$ has low rank — most rows are redundant.

SVD magic. $Y = U\Sigma V^T$ factors into:

  • $U$ — fundamental patterns (eigenimages). Set $C = U$.
  • $\Sigma V^T$ — how each frame mixes the patterns over time. Set $x_t = \Sigma V_t^T$.

Adding the constraint $C^TC = I$ pins down one specific solution out of infinitely many.

State dynamics. Once you have the state sequence $x_t$ (only ~50-dim), fit a linear auto-regression $x_{t+1} = A x_t + \epsilon$ by least squares.

Generate forever. Pick any starting state, push through $A$ with random noise, multiply each new state by $C$, add a touch of image noise — endless new texture frames.

Why SVD is the right tool

SVD is the gold standard for "find the lowest-rank approximation of this matrix" — it's literally optimal in the Frobenius-norm sense. Truncating to the top $n$ singular values gives the best rank-$n$ approximation of your data. For texture, the top-50-ish singular values usually capture 95%+ of the variance.

Identifiability — why the solution isn't unique

You can absorb any invertible matrix $T$ into the state: replace $A \to TAT^{-1}$, $C \to CT^{-1}$, $x_0 \to Tx_0$, and the observations $y$ are unchanged. There's a whole family of equivalent solutions. The constraint $C^TC = I$ picks one canonical representative.

Where this fails

If the texture has any structured object (face, building) the low-rank linear model can't capture it. You'd get blurry blobs. Modern methods use neural GANs/VAEs, which can represent non-linear manifolds.

State-space trajectory — 2-D dynamic texture

drag · play

Each dot is one frame's hidden state $x_t$ in a 2-D state space. The AR rule $x_{t+1} = A x_t + \epsilon$ swirls the trajectory; noise keeps it from being a perfect circle. In a real texture, multiplying $C \cdot x_t$ would render a full image at each step.

Rule of Thumb
  • Pick $n$ (state dim) by keeping enough singular values to capture ≥95% variance. Typically 20–100.
  • Constraint $C^TC = I$ isn't just maths — it picks the canonical solution out of a family of equivalent ones.
  • Both noise sources matter: $v$ keeps the trajectory exploring, $w$ hides the rank. Set $v$ small (a few percent of state magnitude) and $w$ to match real sensor noise.
  • Validate by re-rendering the training clip — if it looks like the original, the model captures the texture; if it looks blurry, increase $n$ or revisit assumptions.
  • Avoid object-textures. Trees waving = OK. A person walking = no.
