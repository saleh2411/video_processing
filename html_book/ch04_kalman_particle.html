Chapter 4

Kalman and Particle Filters

In live video we can't see the future — we have to update an estimate frame by frame. Filtering is the math of "predict + correct": guess where the target will be, then nudge that guess with each new measurement. Kalman is optimal for linear Gaussian worlds; particle filters handle anything else.

Original text (PDF page 28)

Typically, we do not have access to the entire video at once, so we need to process it online. In such situations, filtering techniques are the main tools used. We will first motivate their use through the example of a tracking problem.

Plain-English explanation

"Online" = frames arrive one at a time, no peeking ahead. Filters combine your accumulated belief about the world with each new measurement, producing an updated belief.

4.1 Tracking

Original text (PDF pages 28–30)

Let us assume we have a collection of frames and a motion estimation algorithm. Suppose an external algorithm has detected an object for tracking. The Lucas–Kanade method requires a template to work with — in our case, the content inside the tracking window from the previous frame — as well as a good initial estimate of the object's position in the current frame. In the figure below, the ground truth is marked in red, and the initial guess is marked in blue:

One possible initial guess is the position of the object in the previous frame. When using Lucas–Kanade, we assume that the video has a high enough frame rate so that a few iterations starting from this initial guess will allow the algorithm to converge.

In practice, we do not always have a high frame rate — for example, due to limited CPU performance — and the object may move significantly between frames. In such cases, the initial guess can be far from the ground truth:

In this situation, the Lucas–Kanade method may fail to converge to the correct solution. To address this, we can make assumptions about the motion of the object — assumptions that will later be used by Kalman and Particle filters.

For example, we might assume that the object moves along a straight line, with constant velocity, or with constant acceleration.

Using one of these assumptions, we can predict the object's motion according to the chosen model and refine our initial guess:

In fact, what Lucas–Kanade (LK) does is essentially gradient descent. Suppose this is not what we want. Instead, we may prefer a template matching approach: we take the tracking window from the previous frame, slide it over the current frame, and assign a score at each position based on how well the window matches. In this scenario, we need a Kalman filter for several reasons:

1. Computation efficiency: By predicting the object's position in the current frame, we do not need to perform template matching over the entire frame; instead, we can search within a smaller, localized area:

2. Object ambiguity: There may be multiple objects in the frame that look sinilar — for example, several white cars. In such cases, template matching alone can easily fail to distinguish between them. However, a Kalman filter prediction provides an estimate of the desired object's position, helping to eliminate other candidates.

3. Appearance space: There are cases where an object's appearance may change drastically between frames — for example, due to changes in orientation, lighting, or brightness. In such situations, template matching should not rely solely on the tracking window from the previous frame. Instead, we can create a template that better represents the object's appearance over time. One approach is to average N tracking windows (i.e., compute the average of several images), which is more robust to drastic changes and provides a better representation of the object accumulated over time. Since this involves integrating information over time, using a Kalman filter is particularly effective in this context.

Plain-English explanation

Tracking: some other algorithm picks an object in frame 1, you keep the box on it for the rest of the video. LK works only if the object barely moved and you have a decent starting guess. Three reasons we need a filter on top:

  • Speed: predict → search only near the prediction.
  • Disambiguation: multiple lookalike targets → prediction picks the right one.
  • Appearance over time: build a template that averages many frames; the filter accumulates that information naturally.

The bigger picture — why "filter" at all?

A filter is just a function that takes "what I believed yesterday + what I see today" and outputs "what I believe today." This recursive structure means you don't need to store the whole video — just a small state. That's life-saving for real-time systems.

Three common motion assumptions

  • Constant position (random walk): cheap; assumes object barely moves.
  • Constant velocity: object moves at a steady speed; state = (pos, vel).
  • Constant acceleration: state = (pos, vel, acc). Use for cars / projectiles.

Higher-order = more flexible but more parameters to estimate from noisy data. Same trade-off as in Ch.1.

4.2 Kalman Filter

Original text (PDF pages 30–34)

The following are the main equations of Kalman Filter:

$$ \text{Project: } \hat{x}_{k+1}' = \Phi \hat{x}_k, \quad P'_{k+1} = \Phi P_k \Phi^T + Q $$
$$ \text{Update — Kalman Gain: } K_k = P'_k H^T (H P'_k H^T + R)^{-1} $$
$$ \hat{x}_k = \hat{x}_k' + K_k(Z_k - H \hat{x}_k'), \qquad P_k = (I - K_k H) P'_k $$

Kalman Filter is divided by two stages: project and update.

Project stage

In the project stage, we used all the information of past frames we could accumulate and now we project this information to the new frame. The vector $\hat{x_k}$ has all the information we need (for example the position, velocity, etc). The matrix $\Phi$ predicts the state at time $k + 1$ based on the time $k$.

$P_{K+1}$ is the covariance matrix of x at time k+1.

Update stage

In the update stage, we go over the information received from the current frame and check whether it agrees with the information of past frames. Mathematically, we update the vector $\hat{x_k}$ and it's covarience based on the information of the current frame. This is done by what is called "Kalman Gain". It is the result of the optimal solution we can get based on gaussian assumptions and independence of random variables which will be shown later.

In the tracking example, the initial guess is computed based on past frames. We then apply Lucas–Kanade (LK) or template matching on the current frame to estimate the object's current position. The final result is an integration of the initial guess and the measurement from LK or template matching. If we place more trust in the motion model, the result will be closer to the initial guess; if the current frame provides more reliable information, the result will be closer to the LK or template matching estimate.

Before we start with developing the equations, we will the following notations:
$\hat{x_k}$ - Posterior state vector. This is the final result of Kalman Filter after the processing of frame k.
$\hat{x_k}'$ - Prior state vector. This is the prediction, or the initial guess, based on the past frames.
$\hat{z_k}$ - Measurement vector. This is the output of LK based on the current frame.

Equation Development — Let's assume:

$$ x_{k+1} = \Phi x_k + w_k $$

Where: $x \in R^n$, $\Phi \in R^{n \times n}$, $w \sim N(0, \sigma^2)$.
$x_{k+1}$ is a vector with all the information we want about the world - position, velocity, acceleration, etc. We have a known matrix $\Phi$ which transforms the state $x_k$ to state $x_{k+1}$. $w_k$ is the noise of the model itself. For example, suppose the object we are tracking becomes occluded for 10 frames. We can continue to track it using the motion model's prediction, but without measurements from the current frames, the uncertainty grows over time, and the tracking error accumulates.

Now, let's define the measurement vector:

$$ z_k = H x_k + v_k $$

Where: $z \in R^m$, $H \in R^{m \times n}$, $v \sim N(0, \tilde{\sigma}^2)$.
$x_k$ has all the information of the object (for example, if we have position and velocity then we have 4 parameters). The measurement itself doesn't necessary need all the information. For example, the measurement can be only the position and thus it will have only 2 parameters. H therefore is linear relation between between $x_k$ and $z_k$. $v_k$ is the noise of the measurements )we assume it has normal distribution).

We assume the noise of the process and the noise of the measurements are independent: $E[w_k v_j^T] = 0$.

We will define the System noise as: $Q = E[w_k w_k^T]$

We will define the Measurement noise as: $R = E[v_k v_k^T]$

We will define the error to be: $e_k = x_k - \hat{x_k}$

Where $x_k$ is the ground truth and $\hat{x_k}$ the approximation. Usually, the GT is not given to us, so in Kalman filter we assume the position given in the first frame is the GT (someone gave it to us), and we assume an invariant phenomenon where we try to preserve the optimal position given until now we were at the optimal position.

The covarience matrix would be:

$$ P_k = E[e_k e_k^T] = E[(x_k - \hat{x_k})(x_k - \hat{x_k})^T] $$

Our task is to minimize P (or more accurately, it's eigen values). We emphasize that we want the mean error on the entire frames is minimized, and not just in a single frame. We can visualize this as an elipse of uncertainty, and we want it to be small as possible,

We want: $\hat{x_k} = \hat{x_k}' + K_k(z_k - H \hat{x_k}')$. We insert $z_k = H x_k + v_k$ and get:

$$ \hat{x_k} = \hat{x_k}' + K_k(H x_k + v_k - H \hat{x_k}') $$

We will insert to the definition of P and get:

$$ P_k = E[((I - K_k H)(x_k - \hat{x_k}') - K_k v_k)((I - K_k H)(x_k - \hat{x_k}') - K_k v_k)^T] $$

We simplify the expression and get:

$$ P_k = (I - K_k H) P'_k (I - K_k H)^T + K_k R K_k^T $$

Where $P'_k$ is the covarience before the update, R is the measurement error and $P_k$ is the weighted average of both of them. If K is 0, we don't have measurement error (we don't use the measurement of the current frame). If K is equalt to 1, then the first element will be 0 and we will only have the measurement error.

Kalman suggested finding K which minimize $\text{Trace}[P_k]$. This is because trace is the sum of all the elements in the main diagonal, and in $E[e_k e_k^T]$ those are exactly the variances of each variable in our vector state x. Everything is known to us except K, and this is what we want to find. This is done by differentiating and finding K which makes the derivative 0.

We will get:

$$ T[P_k] = T[P'_k] - 2 T[K_k H P'_k] + T[K_k(H P'_k H^T + R) K_k^T] $$
$$ \frac{\partial T[P_k]}{\partial K_k} = -2(H P'_k)^T + 2 K_k(H P'_k H^T + R) = 0 $$

And we got the Kalman Gain:

$$ K_k = P'_k H^T(H P'_k H^T + R)^{-1} $$

Now we will calculate $P_k$:

$$ P_k = P'_k - P'_k H^T(H P'_k H^T + R)^{-1} H P'_k = P'_k - K_k H P'_k = (I - K_k H) P'_k $$

As shown before. In a private case, let's assume for now that everything is 1-D. We will get in this case:

$$ K = \frac{P'}{P' + R} $$

We can see that when $R \to 0$ (the measurement error goes to 0), then $K \to 1$. In that case, we believe in our measurement and the new state will be solely influenced by the measurement. In other case, if $R \to \infty$, then $K \to 0$ and the new state is only calculated based on the prediction of the model.

By definition: $\hat{x}'_{k+1} = \Phi \hat{x_k}$, $e'_{k+1} = x_{k+1} - \hat{x}'_{k+1} = (\Phi x_k + w_k) - \Phi \hat{x_k} = \Phi e_k + w_k$. We will get:

$$ P'_{k+1} = E[e'_{k+1}(e'_{k+1})^T] = E[(\Phi e_k + w_k)(\Phi e_k + w_k)^T] $$

Let's assume $w_k, e_k$ are independant and we will get:

$$ P'_{k+1} = E[(\Phi e_k)(\Phi e_k)^T] + E[w_k w_k^T] = \Phi P_k \Phi^T + Q $$

The following diagram explains visually how Kalman filter works:

We start with a posterior at time t-1. The x-axis is the state vector x (in this case it's scalar), and the y-axis is the probability of each state vector. We have the deterministic drift which is our $\Phi$. It is the prediction of the current state based on past states. The stochastic diffusion adds error, flattening the gaussian. In addition, we get the measurement of the current time t (the dashed gaussian). Then we incorporate the data of both gaussians to get posterior at time t.

Plain-English explanation — Kalman in one paragraph

You have a noisy guess about the world and noisy measurements. Kalman is the mathematically optimal way to fuse them (assuming Gaussian noise + linear models).

Two alternating stages:

  1. Predict. Push your estimate forward with the motion model $\Phi$. Uncertainty grows (Q).
  2. Update. Mix prediction + measurement via the Kalman gain K: $R \to 0$ ⇒ $K \to 1$ (trust the measurement); $R \to \infty$ ⇒ $K \to 0$ (trust the model). Uncertainty shrinks.

The state vector — what to put in it

Anything you want to track over time. Common combos:

  • $(x,y)$ — pure position. Cheap, fragile to occlusion.
  • $(x, y, \dot x, \dot y)$ — position + velocity. Coasts through brief occlusions.
  • $(x, y, \dot x, \dot y, \ddot x, \ddot y)$ — + acceleration. For cars/projectiles.
  • Add scale, rotation, appearance descriptors as needed. The richer the state, the more measurement you need per frame to keep it well-conditioned.

Why H exists (and isn't always identity)

Your state may contain things you can't directly measure. Example: state = (pos, vel). Your measurement (LK match) gives you pos but not vel. H = [I_2 | 0_2] projects state onto pos. Kalman still estimates velocity, by integrating positions over time.

The covariance ellipse — your visible uncertainty

$P$ is the covariance matrix of the state error. Its eigenvalues/vectors form an ellipse around the estimate showing "I'm pretty sure the truth is in here." Predict step inflates the ellipse (more uncertainty). Update step shrinks it (less uncertainty). Over many steps the ellipse stays bounded if Q and R are reasonable.

Practical gotchas

  • Wrong $\Phi$ → biased estimate. A real car doesn't move at constant velocity forever.
  • Q too small → over-confident in model, ignores measurements, drifts during occlusions or motion changes.
  • R wrong → either too jumpy (R small but data noisy) or too slow (R big but data accurate).
  • Linear-Gaussian assumption doesn't hold (e.g., motion is non-linear, or noise has heavy tails)? Look at Extended/Unscented Kalman, or just use a particle filter.

4.3 Particle Filter

Original text (PDF pages 35–40)

Particle filters wants to do the same as Kalman filter, but without gaussian assumptions. The diagram would be similar:

The distribution on Particle Filter (the red one in the diagram) would no longer be a gaussian. Solving tracking problem with a Particle Filter will be in the following way:

We have a probability distribution over the state space. To represent this distribution, we randomly generate particles — discrete states where the object could be located (typically 100–200 particles). The particles are sampled according to the distribution function, and we need enough particles so that their distribution closely approximates the underlying probability distribution.

During stochastic diffusion, we reduce the probability of each particle to account for uncertainty in the model. Next, we compute the posterior by evaluating how well each particle (tracking window) matches the template. This is done by calculating a matching score for each particle, which measures its similarity to the template. These scores represent the likelihood of each particle.

Then, we compute the posterior by combining the prior and the likelihood. For example, if the model predicts that the object is at a certain tracking window (the prior), but the image in that window does not match the template (the likelihood), then the posterior probability for that particle will be low.

Finally, we select the particle with the highest posterior probability. In the Kalman filter, the maximum of the Gaussian is straightforward to compute—it is simply the mean. In a particle filter, however, determining the most likely state is less direct, and there are several approaches, such as using a weighted average of the particles or selecting the particle with the maximum posterior.

To explain the tradeoff between the maximum and the average, we will use the following figure:

When selecting the particle with the maximum posterior, if two objects have similarly high probabilities, the algorithm may jitter between them, resulting in an unstable estimate. On the other hand, using a weighted average produces a more stable result, but it may sometimes yield an incorrect estimate if the average does not correspond to an actual object location.

Equation development

We will define:
$x_t$ - state vector at time t.
$x_{1:t}$ - $(x_1, ..., x_t)$ the history.
$z_t$ - the measurement at time t.
$z_{1:t}$ - $(z_1, ..., z_t)$ the history of measurements.
One thing we can be interested is the posterior distribution $P(x_{1:t} | z_{1:t})$. However, we will work with filter distribution $P(x_t | z_{1:t}) = \int_{x_1} \cdots \int_{x_{t-1}} P(x_{1:t} | z_{1:t})$.
Lately, we will work with the likelihood:

$$ P(x_{1:t} | z_{1:t}) = \frac{P(z_{1:t} | x_{1:t}) P(x_{1:t})}{P(z_{1:t})} $$

We will use the following assumptions for simplicity:

1. Markov assumption:

$$ P(x_t | x_{1:t}) = P(x_t | x_{t-1}) $$

This assumption was also used in Kalman filter. Therefore:

$$ P(x_{1:t}) = P(x_t | x_{t-1}) P(x_{1:t-1}) = p(x_1) \prod_{j=2}^t P(x_j | x_{j-1}) $$

In the case of Kalman, the matrix $\Phi$ would be $P(x_t | x_{t-1})$.

2. Conditional Independance:

$$ P(z_{1:t} | x_{1:t}) = P(z_t | x_t) P(z_{1:t-1} | x_{1:t-1}) = \prod_{\tau=1}^t P(z_\tau | x_\tau) $$

Now we will develop the equations:

$$ P(x_{1:t} | z_{1:t}) \propto P(z_{1:t} | x_{1:t}) P(x_{1:t}) = \prod_{\tau=1}^t P(z_\tau | x_\tau) \cdot (x_1) \prod_{j=2}^t P(x_j | x_{j-1}) \propto P(z_t | x_t) P(x_t | x_{t-1}) P(x_{1:t-1} | z_{1:t-1}) $$

This expression allows us to describe $P(x_{1:t} | z_{1:t})$ recursively. We can see from here that $P(x_{1:t-1} | z_{1:t-1})$ is the recursion, $P(x_t | x_{t-1})$ is the motion model and $P(z_t | x_t)$ is the likelihood. It can be shown:

$$ P(x_t | z_{1:t}) = \int_{x_1} \cdots \int_{x_{t-1}} P(x_{1:t} | z_{1:t}) = C \cdot P(z_t | x_t) P(x_t | z_{1:t-1}) $$
$$ P(x_t | z_{1:t-1}) = \int_{x_{t-1}} P(x_t | x_{t-1}) P(x_t | z_{1:t-1}) $$

The final equations are therefore:

Posterior:

$$ P(x_{1:t} | z_{1:t}) = B \cdot P(z_t | x_t) P(x_t | x_{t-1}) P(x_{1:t-1} | z_{1:t-1}) $$

Filtering:

$$ P(x_t | z_{1:t}) = \int_{x_1} \cdots \int_{x_{t-1}} P(x_{1:t} | z_{1:t}) = C \cdot P(z_t | x_t) P(x_t | z_{1:t-1}) $$
$$ P(x_t | z_{1:t-1}) = \int_{x_{t-1}} P(x_t | x_{t-1}) P(x_{t-1} | z_{1:t-1}) $$

Now, we will go over the algorithm of Particle filter.

Condensation

Input: $\{S_{t-1}^{(n)}, \Pi_{t-1}^{(n)}, C_{t-1}^{(n)}\}_{n=1}^N$
Output: $\{S_t^{(n)}, \Pi_t^{(n)}, C_t^{(n)}\}_{n=1}^N$
1) Select $S_t^{'(n)}$ as follows: (a) generate r at random r∈[0,1]; (b) find smallest j: $C_{t-1}^{(j)} > r$; (c) Set $S_t^{'(n)} = S_{t-1}^{(j)}$
2) Predict $P(x_t | x_{t-1} = S_t^{(n)})$, i.e. $S_t^{(n)} = A S_t^{'(n)} + B w_t^{(n)}$
3) Measure $\Pi_t^{(n)} = P(Z_t | x_t = S_t^{(n)})$
Normalize: $\sum_n \Pi_t^{(n)} = 1$, and let: $C_t^{(0)} = 0$, $C_t^{(n)} = C_t^{(n-1)} + \Pi_t^{(n)}$
Return $\sum \Pi_t^{(n)} f(S_t^{(n)})$

S - State vector (which was x)
C - slack variable (will be later explained).
$\pi$ - weight of a particle
n - the index of the particle
t - timestep index

At the select stage, we randomly pick particles. $S_t'^{(n)}$ are in fact priors. We will use an example to explain how we randomly sample. Suppose we have 3 particles and we want them to have the following probabilities:

In this case, we desire that we pick the 1st particle 10% of the time, the 2nd particle 60% and the 3rd - 30% of the time. We will do this sampling a random variable r which is has a uniform distribution between 0 and 1. If the $0 < r < 0.1$, then we pick particle 1. If $0.1 < r < 0.1 + 0.6$, then we choose the 2nd particle, otherwise the 3rd particle.

The predict stage is in fact the same as the one in Kalman filter but is more general and we do this for every particle n. In the measure stage, we look at the probability which tells that if the state $x_t$ was $S_t^{(n)}$, then how good was this state for describing the data This is where the similarity function between templates comes in and $\pi$ will indicate the score. An example of could be calculating the Sum of Square Difference (SSD), and then calculate $e^{-SSD}$ to get a value between 0 and 1. The final returned result is a weighted sum.

The following diagram describes the algorithm:

We begin with a weighted set of particles at time t1 and aim to produce a weighted set at time t. The weight of each particle represents its importance: the higher the weight, the higher the probability that the state corresponds to that particle. Next is the resampling stage, where we draw particles deterministically from the original distribution, which is approximated by the current particle set. Particles with large weights may be sampled multiple times, medium-weight particles are sampled once, and very small-weight particles may not be sampled at all. This process is guided by the random variable r described earlier. We then apply drift and diffusion to predict the next state, and finally, we multiply by the likelihood to compute the posterior. The output is the updated weighted particle set, which can be interpreted as a weighted sum representing the state distribution.

Plain-English explanation — particle filter

Kalman assumes Gaussian beliefs. Real beliefs can be multimodal — "the ball is on the table OR under the chair." A Gaussian can't have two peaks.

Particle filter = represent any distribution as a cloud of weighted samples. Each particle is one possible state; its weight is its plausibility. The Condensation loop:

  1. Resample: draw N new particles weighted by old weights — heavy ones get duplicated, light ones die. The cumulative-array + uniform-r trick is a roulette wheel.
  2. Predict: push each particle through the motion model + noise.
  3. Measure: weigh each particle by similarity to the actual measurement (e.g. $e^{-SSD}$).
  4. Normalize. Output = weighted average (smoother) or max-weight (sharper but jittery).

Why "particles" and not just one estimate

A Gaussian has one mean and one variance — that's it. Real beliefs can have multiple modes ("under the desk OR on the table") and weird shapes. Particles are a non-parametric way to represent ANY distribution: just throw down enough dots in proportion to probability, and you've got it. With 200 dots you can approximate any reasonable distribution to good accuracy.

Resampling — the magic and the danger

Resampling concentrates effort where probability is high — dead particles in low-probability regions get pruned. But over-resample and you collapse to a single point ("sample impoverishment"). Standard fix: diffusion (random noise) added on every step keeps the cloud alive.

When to use particle filter over Kalman

  • Non-Gaussian noise (e.g. impulse noise, fat tails).
  • Non-linear dynamics (Kalman needs linearization tricks like EKF/UKF).
  • Multi-modal beliefs (multiple hypotheses about target position).
  • Discrete state spaces.

Cost — be careful

Each particle needs its own measurement evaluation (e.g., template-match score). N particles × M measurement cost = O(NM) per frame. Kalman is O(state³) per frame. For a 4-D state, Kalman wins by orders of magnitude — use particles only when Kalman's assumptions clearly fail.

Takeaways
  • Filtering = predict (motion model) + correct (measurement).
  • Kalman gain K is a "trust dial" — big R → trust model, small R → trust data.
  • Kalman is optimal for linear-Gaussian. Particle filter handles non-Gaussian multimodal worlds.
Rule of Thumb
  • Default to Kalman if your dynamics are roughly linear and noise is roughly Gaussian. It's cheaper and analytically optimal.
  • Switch to Particle Filter when you have multiple hypotheses (lookalike objects) or strongly non-linear dynamics. Start with 100–200 particles; double if estimate looks unstable.
  • Q (process noise) is usually small — say 1–5% of typical step size. Too big = noisy estimate; too small = drift during occlusions.
  • R (measurement noise) ≈ variance of your sensor. Estimate it empirically from a static scene if possible.
  • Use the simplest state that works. Adding velocity costs nothing but accel often over-fits.
  • Always re-initialize if the filter "loses" the target (innovation too large too long). Don't let it drift forever.
  • For particle filters, add diffusion noise every step — this prevents sample impoverishment (all particles collapsing to one point).
